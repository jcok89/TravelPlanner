## ✅ 완성 검수 체크리스트 (AI에게 "체크리스트 검수해줘"라고 시키기)

- [ ] 확정 정보(항공·이동·숙소)가 문서 내 모든 위치에서 1글자도 다르지 않음
- [ ] 여행 날짜의 **요일별** 휴무/제한이 스팟마다 확인됨 (일요일 미사, 월요일 휴관, 공휴일)
- [ ] 야경 일정이 해당 시기 일몰 시각 기준으로 배치됨
- [ ] 모든 연속 일정 구간에 길찾기 버튼 존재, 지도 번호 = 타임라인 번호
- [ ] 예약 필요 항목마다 "언제까지" 시점이 오늘 날짜 기준으로 명시됨
- [ ] 후기 링크는 실접속 검증됨 / 투어는 해당 날짜 예약 가능 확인됨
- [ ] research/ 폴더에 조사일·출처 포함 기록 5종 완료
- [ ] 루트 허브 카드 추가 + git push 완료

## 📦 다른 사람에게 공유할 때 — 파일 패키지

| 필수 | 파일/폴더 | 용도 |
|---|---|---|
| ✅ | `README.md` | 처음 받은 사람용 진입점 (30초 시작 안내) |
| ✅ | `여행계획_AI작업가이드.md` | 전체 방법론 + 복붙 프롬프트 (이것만 있어도 재현 가능) |
| ✅ | `새여행_시작_템플릿.md` (이 파일) | 빈칸 채우기식 스타터 |
| ✅ | `.agents/` 폴더 전체 | AGENTS.md + 스킬 3종 (myrealtrip MCP 스크립트 포함) — Codex/Gemini용 |
| ✅ | `.claude/` 폴더 전체 | Claude Code에서 스킬 자동 인식용 (내용은 .agents/skills와 동일) |
| 권장 | `CLAUDE.md` | Claude Code용 규칙 포인터 |
| 권장 | `index.html` | 여행 목록 허브 (비어 있는 상태로 시작) |
| 권장 | `server.py` | 완성된 계획서 로컬 미리보기 (`python server.py`) |
| 선택 | `requirements.txt` | myrealtrip 투어 조회용 의존성(mcp) |

한 번에 압축하려면:

```bash
zip -r travel-ai-kit.zip README.md CLAUDE.md 여행계획_AI작업가이드.md 새여행_시작_템플릿.md \
  index.html server.py requirements.txt .agents .claude
```

받은 사람은 압축을 풀고 → 폴더에서 AI 에이전트 실행 → 이 템플릿의 스타터 프롬프트만 채워서 붙여넣으면 끝.
