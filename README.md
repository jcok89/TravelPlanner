# ✈️ AI 여행 계획 키트 (Travel Planner Kit)

AI 에이전트(Claude Code · Codex · Gemini CLI 등)로 **"빈틈없는 인터랙티브 여행 계획서"** 를
반복 재현할 수 있게 만든 배포용 스킬 키트입니다.

결과물은 이런 단일 HTML 계획서입니다 — **날짜별 탭 + 타임라인 + 지도(Leaflet) + 구간별 구글맵 길찾기
+ 선택지 저장(localStorage) + 예약 우선순위 + 실접속 검증된 후기 링크 + 날짜별 예약가능 확인된 투어**.
서버 없이 브라우저로 바로 열립니다.

> 이 키트는 실제 사례(체코·오스트리아·헝가리 6박 8일 계획서) 제작에 쓴 스킬과 프롬프트를
> 그대로 재사용 가능하게 뽑아낸 것입니다. **여기엔 스킬·가이드만 담겨 있고, 완성 여행 예시는 포함하지 않습니다.**

---

## 🚀 30초 시작 (가장 빠른 길)

1. AI 에이전트를 이 폴더에서 실행합니다 (예: `claude` 또는 `codex`).
2. `새여행_시작_템플릿.md`를 열어 **스타터 프롬프트**를 복사합니다.
3. `{ }` 빈칸(기간·항공·숙소·여행 스타일)을 채워 AI에 붙여넣습니다.
4. AI가 리서치 → `index.html` 계획서 생성 → 허브 카드 추가까지 자동으로 진행합니다.
5. 완성된 계획서를 보려면: `python server.py` 실행 후 브라우저에서 `http://localhost:8999`.

그게 전부입니다. 아래는 각 부분이 무엇인지에 대한 설명입니다.

---

## 📂 폴더 구성

```
Travel Planner/
├── README.md                     ← (지금 이 파일) 진입점
├── CLAUDE.md                     ← Claude Code용 규칙 포인터 (→ .agents/AGENTS.md)
├── 여행계획_AI작업가이드.md        ← 전체 방법론 + 단계별 복붙 프롬프트 (가장 중요)
├── 새여행_시작_템플릿.md           ← 빈칸 채우기식 스타터 프롬프트 + 검수 체크리스트
├── 예시_테스트케이스_가이드.md      ← 키트가 실제로 동작함을 보인 회귀 테스트 + 비교분석 (따라 하기 예시)
├── 예시_202610_오사카교토/          ← 위 테스트의 완성 산출물 (가상 데이터 · 레퍼런스 구현)
├── index.html                    ← 여행 목록 허브 (완성한 여행 카드가 여기 쌓임)
├── server.py                     ← 로컬 미리보기 서버 (python server.py)
├── requirements.txt              ← myrealtrip 스킬용 의존성(mcp). 나머지엔 불필요
├── .agents/                      ← 모든 AI 도구 공통
│   ├── AGENTS.md                 ←   워크스페이스 규칙 + 스킬 표 (단일 소스)
│   └── skills/
│       ├── travel-research/      ←   3-트랙 병렬 리서치 방법론
│       ├── travel-plan-builder/  ←   단일 HTML 계획서 제작 표준
│       └── myrealtrip-search/    ←   마이리얼트립 공식 MCP 투어 조회 (+ 파이썬 래퍼)
└── .claude/skills/               ← Claude Code 자동 인식용 (내용은 .agents/skills와 동일)
```

새 여행을 만들면 여기에 `YYYYMM_지역명/` 폴더가 생기고, 그 안에 `index.html` · `travel_pins.kml` · `research/*.md`가 채워집니다.

## 🧩 핵심 스킬 3종

| 스킬 | 역할 | 언제 도는가 |
|---|---|---|
| **travel-research** | 여행사 패키지 / 실제 후기 / 공식 운영정보 **3-트랙 교차검증** | "여행 리서치", "필수 코스 검증", "운영시간 확인" |
| **travel-plan-builder** | 검증된 재료 → **단일 HTML 인터랙티브 계획서** (지도·길찾기·선택저장·KML) | "계획서 만들어/개선", "일정 UI" |
| **myrealtrip-search** | 마이리얼트립 공식 MCP로 **날짜별 예약가능 투어** 조회 | "마이리얼트립 투어 찾아줘", "우리 날짜에 예약되는지 확인" |

스킬 문서는 도구가 자동으로 읽습니다. 사람이 직접 절차를 보고 싶으면 `.agents/skills/<이름>/SKILL.md`를 여세요.

## 🔑 이 방식이 지키는 5대 원칙

1. **확정 정보(항공·기차·숙소·확정 예약)는 절대 불변** — AI가 시간/역/주소를 임의로 못 바꾸게 프롬프트에 못박습니다.
2. **교차검증** — 패키지(업계 검증) × 후기(만족도) × 공식 사이트(우리 날짜 운영 여부) 3개가 맞아야 확정.
3. **링크는 실접속 검증** — 제목만 보고 넣지 않습니다 (실제로 15개 중 4개가 제목-내용 불일치였음).
4. **날짜별 예약 가능까지 확인** — 평점 높은 투어여도 우리 날짜에 안 되면 무의미.
5. **모든 조사는 MD로 기록 + git** — 조사일·출처 URL을 남겨 나중에 재검증 가능하게.

자세한 배경과 실전 사례는 `여행계획_AI작업가이드.md`에 있습니다.

## 🛠️ 사전 준비

| 항목 | 필요 시점 | 방법 |
|---|---|---|
| **웹검색+파일작성 되는 AI 에이전트** | 항상 | Claude Code / Codex / Gemini CLI 등 |
| **Python 3** | 로컬 미리보기, myrealtrip 조회 | 시스템에 설치 |
| **`mcp` 파이썬 패키지** | myrealtrip 투어 조회할 때만 | `python -m pip install -r requirements.txt` |
| **마이리얼트립 MCP** | 투어 조회할 때만 | 별도 API키 불필요 (공식 공개 엔드포인트). 스킬 래퍼가 직접 호출 |

> Windows에서 파이썬 한글이 깨지면: `export PYTHONIOENCODING=utf-8`

이 폴더를 통째로 압축하면 됩니다:

```bash
zip -r travel-ai-kit.zip README.md CLAUDE.md 여행계획_AI작업가이드.md 새여행_시작_템플릿.md \
  index.html server.py requirements.txt .agents .claude
```

받은 사람은 압축을 풀고 → 폴더에서 AI 에이전트를 켜고 → 위 **"30초 시작"** 대로 하면 됩니다.

## ✅ 완성 검수 체크리스트

여행을 하나 만든 뒤 AI에게 "체크리스트 검수해줘"라고 시키면 아래를 점검합니다.

- [ ] 확정 정보(항공·이동·숙소)가 문서 내 모든 위치에서 1글자도 다르지 않음
- [ ] 여행 날짜의 **요일별** 휴무/제한이 스팟마다 확인됨 (일요일 미사, 월요일 휴관, 공휴일)
- [ ] 야경 일정이 해당 시기 일몰 시각 기준으로 배치됨
- [ ] 모든 연속 일정 구간에 길찾기 버튼 존재, 지도 번호 = 타임라인 번호
- [ ] 예약 필요 항목마다 "언제까지" 시점이 오늘 날짜 기준으로 명시됨
- [ ] 후기 링크는 실접속 검증됨 / 투어는 해당 날짜 예약 가능 확인됨
- [ ] `research/` 폴더에 조사일·출처 포함 기록 완료
- [ ] 루트 허브(`index.html`)에 카드 추가 + git push 완료

## 🧷 자주 겪는 문제

| 문제 | 해결 |
|---|---|
| 여행사 사이트(SPA)가 안 열림 | 공식 블로그·검색 스니펫·유사 패키지 일정표로 대체 |
| 네이버 블로그 크롤러 차단 | 브라우저 UA로 HTTP 요청 → 200 + 제목/키워드 확인 |
| 파이썬 한글 깨짐(Windows) | `export PYTHONIOENCODING=utf-8` |
| 숨긴 탭의 Leaflet 지도 깨짐 | 탭 첫 표시 때 초기화(lazy) + 재표시 때 `invalidateSize()` |
| 같은 정보가 문서 곳곳에서 어긋남 | 데이터(JS 배열) → 렌더 함수 구조로 단일 소스화 |
