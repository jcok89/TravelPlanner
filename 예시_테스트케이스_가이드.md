## 2. 실제로 실행된 파이프라인 (스킬대로)

| 단계 | 스킬 | 실행 방식 | 실제 산출 |

### 이 테스트에서 실제로 "잡아낸" 것 (스킬이 값을 하는 증거)
1. **텐류지 법당 "운룡도"는 토·일·공휴일만 공개** → 아라시야마를 **일요일(10/25)** 로 배치. (가이드북만 보면 놓칠 요일 함정)
2. **교토 단풍 야간 라이트업은 11월 중순 이후 시작** → 10월 하순 일정엔 **없음**. "야경=우메다·도톤보리로 대체"로 수정.
3. **일몰 17:16→17:13** 확인 → 우메다 공중정원 **16:30 입장**을 골든타임으로 명시.
4. **오사카 주유패스 1일권(3,500엔)** = 오사카성+우메다 같은 날이면 본전 이상 → 두 곳을 10/22 하루에 몰기.
5. **간사이 스루패스 2일권 유효기간이 10/1까지**로 안내됨 → 우리 일정 사용 불가 가능성, "하반기판 확인" 경고.
6. **트랙4 링크검증: 9개 중 1개 404**(트립닷컴 맛집) → 계획서엔 실접속 200 링크만 게재.
7. **마이리얼트립 교토 기모노 체험(gid 3828614)이 10/24 예약가능** 확인 → 일정에 채택. (오사카 야경 프라이빗은 고베 동선 과함 → 대안 카드로만)

## 4. 개선사항 (테스트에서 발견 → 조치)

| # | 발견 | 조치 |
|---|---|---|
| 1 | 키트에서 원본 예시를 들어내며 **참고 구현이 사라짐**. travel-plan-builder SKILL이 없는 파일을 가리키고 있었음 | ✅ 이 예시(`예시_202610_오사카교토/`)를 **번들 레퍼런스로 포함**하고 SKILL·README가 이 폴더를 가리키도록 수정 |
| 2 | 투어를 일정에 촘촘히 녹이는 패턴이 스킬에 약하게만 기술됨 | ✅ travel-plan-builder SKILL에 **"날짜별 예약가능 확인된 투어를 stop 또는 alts 카드로 임베드"** 규칙 보강 |
| 3 | 해외 여행은 **가격 환율 병기** 기준이 필요(트랙 C가 100엔≈930원 가정을 스스로 명시) | ✅ travel-research SKILL 트랙 C 산출물 규칙에 **"환율 가정 명시 + 결제화면 기준"** 문구 추가 |
| 4 | 링크검증 단계(트랙 4)가 가이드엔 있으나 **실제 404 발생률**이 체감 안 됐음 | ✅ 이 테스트에서 **9개 중 1개 404** 실측 → 트랙 4의 필요성 데이터로 문서화(`research/04`) |
| 5 | 요일별 운영 함정(운룡도 토·일만, 라이트업 미개최)이 **정확히 일정에 반영되는지** 미검증이었음 | ✅ 실제로 일요일 배치/야경 대체가 반영됨을 확인 — 방법론 유효성 입증 |

> 위 1~3은 스킬/문서에 실제 반영했습니다(아래 §6 재현 절차와 함께 커밋).
