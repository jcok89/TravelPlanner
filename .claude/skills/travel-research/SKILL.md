# 여행 3-트랙 병렬 리서치

## 이 스킬이 하는 일

여행 일정을 "감"이 아니라 **3개의 독립된 정보원**으로 교차검증한다.
각 트랙은 독립적으로 조사 가능하므로, 서브에이전트/병렬 세션을 지원하는 도구(Claude Code Agent, Codex 멀티태스크 등)에서는 **반드시 3개를 동시에 실행**한다. 지원하지 않으면 순차 실행한다.

```
사용자 요청 (도시/날짜/여행 스타일)
  ├─ 트랙 A: 메이저 여행사 패키지 분석  → "업계가 검증한 필수 코스"
  ├─ 트랙 B: 실제 여행자 후기 검증      → "진짜 좋은 곳 / 실망 스팟"
  └─ 트랙 C: 공식 운영정보 실무 검증    → "우리 날짜에 실제로 가능한가"
  → 교차검증 종합 → 일정 설계 재료 완성
```

## 트랙별 조사 지침

### 트랙 A — 여행사 패키지 분석 (하나투어·모두투어 등)

목적: 패키지가 공통으로 넣는 스팟 = 업계 검증 필수 코스. 패키지가 "선택관광"으로 파는 것 = 비용 들여도 갈 가치가 있다고 팔리는 것.

- 검색어: `"{여행사} {지역} 일정표"`, `"{지역} 패키지 선택관광"`, `site:hanatour.com`, `site:modetour.com`
- 여행사 사이트는 동적(SPA) 페이지가 많아 직접 수집이 안 되면 **공식 블로그(하나투어스토리 등)·검색엔진 스니펫·유사 패키지(삼호관광 등) 일정표로 대체**
- 산출물: 도시별 표 `관광포인트 | A사 포함여부 | B사 포함여부 | 기본/선택 | 옵션가격`, 근교도시 옵션과 이동시간, 후기의 만족/불만 포인트

### 트랙 B — 실제 여행자 후기 검증

목적: 후보 스팟 목록을 주고 "언급도"와 "실망 후기"로 순위를 매긴다.

- 소스: 네이버 블로그(최신순), 티스토리, 트립어드바이저, 레딧(r/travel, 도시별 서브), 마이리얼트립 커뮤니티, 현지 가이드 블로그(Honest Guide 등)
- 반드시 물어볼 것: ① 필수 TOP 순위와 추천 시간대 ② "가봤는데 별로" 스팟 ③ 계절 팁(일몰 시각, 더위/추위, 혼잡 시간대) ④ 여행 스타일별 경험(커플=야경/로맨틱 등) ⑤ 맛집 웨이팅/예약 팁
- 산출물: 스팟별 `언급도 상/중/하 | 추천 시간대 | 한 줄 팁` 표, 제외 추천 목록과 이유, 일몰 시각표

### 트랙 C — 공식 운영정보 실무 검증 (가장 중요!)

목적: **우리 여행 날짜의 요일**에 각 스팟이 실제로 운영하는지, 예약은 언제 열리고 언제 매진되는지 확인. 여행 가이드북 정보는 자주 낡는다(예: 빈 48시간권은 2026년 폐지됨).

- 각 스팟의 **공식 사이트**에서: 개장시간(요일별!), 티켓 종류/가격, 시간지정 예약 필요 여부, 예약 오픈 시기, 매진 속도
- 요일 이슈 필수 확인: 일요일 미사(성당류), 월요일 휴관(박물관류), 토요일 샤바트(유대인 지구), 국경일
- 교통: 도시별 관광객 패스 최신 요금, 공항 이동 수단
- 산출물: 항목별 `확인된 사실 | 우리 일정에 미치는 영향 | 권장 조치(예약 시점)` + **예약 우선순위 표(오늘 날짜 기준 긴급도)**
- 해외는 가격을 **현지통화+원화 병기**하고 문서 상단에 **환율 가정(예: 100엔≈930원, 조사일 기준)** 과 "실제 결제는 카드사 환율" 단서를 명시한다

## 교차검증 규칙

1. **패키지 포함(A) + 후기 언급도 상(B) + 운영 확인(C)** = 확정 필수 스팟
2. A에는 있는데 B에서 혹평 → 기대치 하향 or 동선 끼워넣기로 격하 (예: 천문시계 정각 쇼)
3. A에 없는데 B에서 극찬 → 자유여행만의 차별 포인트로 채택 (예: 세체니 온천, 국회 내부투어)
4. B/C가 상충하면 **C(공식·최신)가 우선**. 가격이 소스 간 다르면 "결제화면 기준 확인 필요"로 표기
5. 모든 결론에 출처 URL을 남긴다

## 산출물 형식

**리서치 원문은 반드시 여행 폴더의 `research/` 디렉터리에 MD 파일로 보존한다** (대화에만 남기지 말 것 — 나중에 재검토/갱신 가능해야 함):

```
{여행폴더}/research/
├── README.md                          # 조사일 + 교차검증 최종 결론 요약
├── 01_패키지비교_{여행사들}.md          # 트랙 A 원문
├── 02_실제후기_필수스팟검증.md          # 트랙 B 원문
└── 03_운영시간_예약_실무검증.md         # 트랙 C 원문
```

각 파일 상단에 조사일과 조사 방법을 인용구(`>`)로 명시하고, 하단에 출처 URL을 남긴다. 작성 후 git 커밋·푸시한다.

최종 보고는 다음 섹션으로 종합한다:
- 도시별 필수 스팟 순위표 (근거: 패키지 포함여부 + 후기 언급도)
- 제외/기대치 하향 스팟과 이유
- 예약 우선순위 표 (오늘 기준: "지금 즉시 / N주 내 / 출발 전 아무 때나")
- 계절 팁 (일몰 골든타임, 더위/혼잡/치안)
- 이 재료는 `travel-plan-builder` 스킬의 입력이 된다

## 완료 기준

- 3개 트랙 모두 실행하고 출처 URL을 확보했다
- 리서치 원문을 `research/*.md`로 저장하고 커밋·푸시했다
- 우리 여행 날짜의 **요일별 휴무/제한**을 스팟마다 확인했다
- 예약 필요 항목마다 "언제까지 예약해야 하는지"를 오늘 날짜 기준으로 명시했다
- 후기 혹평 스팟을 그대로 일정에 넣지 않고 격하/제외 판단을 기록했다
